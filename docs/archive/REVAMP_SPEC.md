# 大改版 spec — 胶圈 tray / 胶片缩小 / 单观片台 / 卷类型

> 背景：`index.html`（单文件、原生 HTML/CSS/Canvas、零依赖）。这一组是结构性改动（tray 交互、lightbox 尺度、观片台模型、新建卷类型），区别于 `REALISM_SPEC.md` 的逐帧拟真细节。
> 约束：**纯原生、零依赖、单文件**，复用现有 `addPiece`/拖出机制、`pieceLayout` 几何、`renderPieceFilm`、`presets`、`?selftest`。
> 依赖/前情：阶段0（PR #5）把"透光感"改名"背光亮度"、挪进观片台组、片基半透明——但它把背光做成提亮胶片（错，见阶段 A）。**PR #5 废弃不合**，其唯一可留的「片基恒定半透明」并入阶段 A 重做。本组从干净 main（不含 PR #5）起步。
> **四个阶段都改同一文件 `index.html` 同一渲染/交互路径，不可并行,必须串行**（参考 REFACTOR_SPEC 同文件串行约定）。建议顺序 A→B→C→D，各自独立 PR。

## 阶段 A — 观片台精简：只留「专业观片台」，设置只留「背光亮度」

只保留一台观片台，砍掉切换与多余微调。

0. **并入阶段0 的片基半透明**（PR #5 废弃，这里重做）：`renderPieceFilm` 片基 alpha 固定为半透明常量（让背后台面透上来、边框始终通透），不随任何旋钮变。
1. **删除台子切换**：底部圆点 `dots`、抽屉 `presetSeg` 段、左右轮播 `go/jump` 动画与相关键位，全部移除。新增「背光亮度」控件到观片台设置组（原 glow 重命名，语义见第 4 点）。
2. **硬编码单台** = 专业观片台（`base:#ffffff, unif:0, pixel:0, plate:'FILMSCAN PRO'`），`presets[]` 退化为单一固定配置（或直接内联常量）。
3. **观片台设置组只留「背光亮度」**：去掉 均匀度(unif)/像素网格(pixel)/底色(base swatches) 三个控件与其 UI。
4. **「背光亮度」= 只调观片台面板亮度，不碰胶片像素（重要，阶段0 做反了）**：
   - 理由：胶片是自由摆在桌面上的，可不在观片台上。若背光提亮所有胶片，则**摆到台外的胶片也会变亮 = 荒谬**。背光只是台子那盏灯，胶片是"设置好什么样就什么样"。
   - `renderBg`：面板（#bg，= lightbox = 导出窗）亮度 = f(背光亮度)。低 → 暗面板，高 → 亮白面板。
   - **删掉阶段0 加在每帧上的 `lighter` 提亮**——背光不直接改胶片像素。
   - 胶片照旧渲染（**片基半透明**是关键）。其半透明部位（片基/齿孔）在 DOM 图层合成 + 导出合成中**自然透出背后的东西**：压在观片台上 → 透出亮面板（看着被背光照亮）；摆到台外桌面 → 透出暗桌面（不被照亮）。于是「台外胶片不被背光提亮」**靠合成白拿、无需位置判断**。
   - 照片（不透明乳剂）始终 WYSIWYG，背光不改其明暗。
5. 向后兼容、`?selftest` 不破、导出正常（面板被调暗时导出底色随之变暗属预期）。

验收：界面无任何台子切换入口；设置抽屉观片台组只剩「背光亮度」一项；**拖背光亮度时只有观片台面板明暗变化** + 压在台上的胶片其半透明片基/齿孔随面板透光变亮变暗；**摆到台外的胶片完全不受背光影响**；照片本身始终不变。

## 阶段 B — 胶片缩小（lightbox 高度内约能竖排 7 条胶卷带）

胶片现在太大。以「真实观片台垂直能放约 7 卷」为尺度参照缩小（**仅尺寸参照，piece 仍自由摆放，不做自动竖排布局**）。

1. `pieceLayout` 里帧高 `fh` 缩小：目标让导出窗（`TW×TH=2000×1500`）**垂直方向目测能并排约 7 条** 135 胶卷带（含间距）。当前 `fh=TH*(0.22+ratio*0.34)`（135 时 ≈540px，带高 BH 更大，竖排放不下 3 条）。
2. 按「7 条竖排」反推：单条带高 `BH≈TH/7≈214`，留间距则 `fh` 更小；开发者按此目标实测调 `fh` 比例（量级约 `fh=TH*(0.08~0.12+ratio*0.12)`，以实际能竖排约 7 条 135 为准）。
3. 单张/拍立得/无边框跟随 `fh`，无需单独处理。

验收：lightbox 高度内目测能竖排约 7 条 135 胶卷带；胶片在台面留白明显变多。

## 阶段 C — 右侧 tray 胶圈化 + hover 向左水平展开预览

把 tray 里每卷从竖直缩略图列表，改成「卷起的胶圈」，hover 才铺开。

1. **默认胶圈态**：tray 里每卷显示成「卷起的胶圈/胶卷盘」小图（CSS/Canvas/SVG 画的 coil，纯原生，不引资源依赖），占位小，竖排能并列多卷。
2. **hover 向左水平展开预览**：hover 某卷 → 向左（朝台面方向）**水平**铺开一条该卷胶片带预览（帧横向排开；**注意是水平，不是原来的垂直缩略图列**）。移开即收。**展开仅预览**。
3. **拖拽语义不变**：仍从该卷（胶圈或展开预览区）`pointerdown` **整卷**拖到台面，沿用现有 `addPiece`/拖出机制。展开**不**引入拖单帧——还是抓整卷。
4. tray 其余交互（新建卷、卷内导入「＋」、删除卷）保持可用。

验收：tray 默认是一列胶圈；hover 某卷向左水平展开帧预览、移开收起；从卷拖出整卷到台面照常工作。

## 阶段 D — 新建卷选胶片类型（反转 / 黑白 / 负片）+ 按类型渲染

> ⚠️ 本阶段**推翻 `REALISM_SPEC.md` 原「只走正片」锁**：负片（反相+橙罩）作为 per-roll 选项重新进入。

1. **新建卷 UI**：新建卷时可选三类型 **反转(reversal) / 黑白(bw) / 负片(negative)**。
2. **roll 增 `filmType` 属性**（`"reversal"|"bw"|"negative"`），默认 `reversal`（=现状正片观感），旧 roll 无字段按 `reversal`（向后兼容）。
3. **按 `filmType` 渲染**（在 `renderPieceFilm` 逐帧 cover 之后、clip 内处理，保持透光/背光叠加顺序正确）：
   - **reversal**：正常照片（现状）；片基=现状清透灰半透明。
   - **bw**：画面灰度化（`filter='grayscale(1)'` 或手算亮度）；片基灰。
   - **negative**：画面**反相**（invert RGB）**+ 橙色蒙版**叠加（C-41 橙罩观感）；**片基也是橙色**（区别于 reversal/bw 的清灰）。
4. **切类型实时重渲**：改 roll 类型后该卷所有 piece 重渲。
5. 与透光/背光：负片片基橙、反转/黑白片基清灰——和阶段0「片基恒定半透明」叠加（橙/灰都是半透明，背光照样透上来）。

验收：新建反转/黑白/负片三种，画面分别 正常 / 灰度 / 反相+橙罩；切类型实时变；导出三种都正确。
