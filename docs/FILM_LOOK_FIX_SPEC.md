# 胶片质感三轮修正 — 双层重构 + 观感修复

> 背景：FILM_LOOK_SPEC（10 阶段）已落 main，但对照实物有 8 处反馈。本轮先把渲染从「单 canvas 摊平」重构成「胶卷背景层 + 照片层」双层（用户要的心智模型），再在新结构上逐条修。
> 约束：纯原生 TS + Canvas，零运行时框架。胶片渲染在 `src/render.ts`，tray 展开在 `src/tray.ts` + `src/styles.css`，导出在 `src/presets.ts`。
> 各阶段独立 PR；**1 是地基必须最先；2–7 同改 `render.ts` 渲染路径必须串行；8 改 tray/styles 独立文件**。建立在当前 main（10 阶段已落）之上。

## 决策（已和用户敲定 2026-06-05）

- **#7 图层**：重构成双层——胶卷背景（片基/齿孔/印字/漏光/落影）一层，照片+逐帧特效一层叠上。
- **#8 halation**：按真实高光门控 + 大幅减弱（采样画面亮度，只在真高光区出极淡红橙，强度砍到现在 ~1/3）。撤掉现在「每帧无条件在上中糊一团红」。
- **#4 纯黑边框**：仅反转/黑白片基改纯黑 `#000`；**负片保留 C-41 橙基**。
- **#1 微倾斜**：针对**整卷**，不是剪下的单张（现在条件反了）。
- **#2 印字**：条码/帧号/厂牌/日期现在压在齿孔上 → 移到齿孔**外的独立窄边**，不压孔。
- **#5 负片融合**：负片的帧和橙基是两套橙硬切、有突兀缝 → 统一橙 + 边缘羽化，融成一整条。
- **#6 漏光**：现在只在片头片尾对称糊两竖条、太刻意 → 改确定性但有机错位形态（斜向/单侧/位置错开）。
- **#3 展开动画**：hover 现在是胶片从上方落下 → 改成从暗盒侧**横向抽出卷开**（暗盒在右，胶片向左拉出）。

确定性约束（贯穿全程）：所有「随机感」效果一律用 LCG 按 `piece.id`/帧索引派生，**不用 `Math.random`**，保证 `?selftest` 可复现。

## 阶段 1 — 双层地基重构（foundation，必须最先落，本阶段零观感变化）

把每个 film piece 从单 canvas 拆成两块**叠放** canvas：

1. **piece 结构**：`piece.el` 内含 `baseCanvas`（下）+ `photoCanvas`（上），二者 `position:absolute`、同尺寸（`L.cw×L.ch`）、同 `deckScale`、同定位完全重合。改 `types.ts`（`Piece` 加 `photoCanvas`/`photoCtx`，原 `canvas`/`ctx` 作 base）、`pieces.ts`（创建时建两块并塞进 `el`）、`deck.ts`/`main.ts` 里任何引用 `piece.canvas` 取尺寸/定位的点同步。
2. **render 拆分**：`renderPieceFilm` 拆成
   - `renderPieceFilmBase(piece)` → 画 base：落影 + 实色片基 + 齿孔镂空 + 边缘印字 + 漏光。
   - `renderPieceFilmPhotos(piece)` → 画 photo：逐帧 cover 照片 + 暗角 + halation + 密度微差 + 灰尘毛发 + 负片处理 + 选帧虚线框。
   - 两者共用 `pieceLayout(piece)` 同一份几何（cw/ch/originX/framesY…）。
3. **bare/polaroid 仍单层**：画进 `baseCanvas`，`photoCanvas` 留空（透明）。渲染路由 `renderPiece` 按 style 分发不变。
4. **导出同步（关键坑）**：`presets.ts` `save` 现在 `oc.drawImage(pc.canvas,...)` 只画一层 → 改成**先 base 再 photo** 两块都画（同 `pc.x/pc.y`、同 `pc.rotation` 旋转）。否则导出丢照片层。
5. 向后兼容、`?selftest` 不破。

验收：**视觉和重构前逐像素基本一致**（纯结构改、本阶段不许动任何观感参数）；拖拽/选帧/剪下/导出/`?selftest` 全部正常；导出图包含照片（不只剩片基）。

## 阶段 2 — 微倾斜改针对整卷（#1）

1. `render.ts` 末尾旋转逻辑条件反过来：**整卷**（`piece.shots === null`）给 ±3° 确定性倾斜（角度按 `piece.id` 派生）；剪下的单张（`shots !== null`）**不倾斜**（`transform=''`、`rotation=0`）。
2. `piece.rotation` 同步设值；导出已读 `pc.rotation`，自动跟随。

验收：整条胶卷略带随手倾斜；剪下的单张摆正；拖拽与导出旋转一致、不破。

## 阶段 3 — halation 高光门控 + 大幅减弱（#8）

1. photo 层：撤掉现在「每帧无条件在 `fw*0.50,fh*0.30` 用 lighter 糊红橙渐变」的写法。
2. 改成**按真实高光门控**：对每帧 drawImage 后 `getImageData` 采样亮度（同源不污染），只在亮度超阈值的高光像素**邻域边缘**叠极淡红橙晕；暗部/中灰几乎不变。
3. 强度砍到约现在的 1/3（`HALATION_STRENGTH` ~0.06）。采样是确定性的，`?selftest` 可复现。

验收：纯亮天空/高光边缘见微弱红橙晕；暗帧/无高光帧**不再有中间红疙瘩**；自检不破。

## 阶段 4 — 负片帧↔橙基融合（#5）

1. 负片现状：基是橙 `200,112,42@0.92`，帧是 `invert(1)` + 另一支橙 `200,100,30@0.35` 两套橙叠、边界硬切。
2. 统一成**同一支 C-41 橙**（基与帧蒙版用同一组 RGB/密度），并让帧画面边缘**羽化过渡**到基色（如帧外缘一圈渐隐到基橙），消除帧与基之间的突兀硬缝。
3. 反转/黑白不受影响。

验收：负片的帧和橙基无可见硬边界、像一整条连续胶片；反转/黑白照旧。

## 阶段 5 — 边缘印字移出齿孔（#2）

1. 现状：齿孔带占 `bandTop+BH-m*0.72 ~ -m*0.30`，印字糊在 `BH-m*0.75` 正好压齿孔上。
2. 重排片基边距/齿孔/印字的 y 分配：齿孔一行 + 其外侧（更贴片边）**独立窄印字带**；条码/帧号/厂牌/日期落在该窄带，**不与齿孔重叠**。必要时 `m`（边距带）略增以容纳印字窄边。
3. 印字随 `fh` 缩放、长卷帧号递增照旧。

验收：条码/帧号/厂牌/日期在齿孔之外的窄边上，不再压在孔里；齿孔排布正常、不出界。

## 阶段 6 — 反转/黑白片基纯黑（#4）

1. base 层片基色：非负片从近黑 `30,26,22` → **纯黑 `0,0,0`**（`#000`）。
2. 负片保留 C-41 橙不变。

验收：反转/黑白卷片基纯黑；负片卷仍橙。

## 阶段 7 — 漏光有机化（#6）

1. base/overlay 层：漏光从「片头片尾对称两竖条」改成**确定性但有机错位**形态——斜向条带 / 单侧偏置 / 位置与角度按 `piece.id` 派生错开、形状不规则，不再机械对称糊两端。
2. 仍受全局开关控制、默认关。LCG 确定性、不破 `?selftest`。

验收：开启后漏光像随手撒上、非对称机械两端；关闭则无；自检不破。

## 阶段 8 — hover 展开改横向卷开（#3）

1. `tray.ts` + `styles.css`：`#rollExpand` 展开动画从现在「容器+子帧 `translateY(-12/-8px)→0` 自上落下」改成**从暗盒侧横向抽出**：容器/子帧用 `translateX` 或 `clip-path` 横向展开，逐帧**左→右级联抽出**，方向贴合「暗盒在右、胶片向左拉出卷开」。
2. 入场/收起对称、`scheduleHideExpand` 节流逻辑保持。

验收：hover 卷时胶片从暗盒方向横向卷出铺开（非自上而下落）；移开收回正常。
