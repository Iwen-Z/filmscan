# 观片台重构 spec — 自由摆放的胶片台面

> 背景：当前 `index.html`（单文件，原生 HTML/CSS/Canvas）的模型是「一卷 = 铺满台面的一整张 `#film` canvas，只能左右 `panX` 平移，同一时刻只显示 activeRoll 一卷」。
> 目标：升级成「台面上可同时摆多片胶片、每片可在 2D 上自由拖动（拼图手感）」，并借这次重构落地 4 个需求。
> 约束：**保持纯原生、零依赖、单文件 `index.html`**。不引入框架/构建链/canvas 库。复用现有 `films[]` 规格、`stripLayout()` 几何、`renderBg()`、逐帧 cover 裁切、`pointerdown/move/up + setPointerCapture` 拖动模式。

## 目标模型（术语）

- **台面（deck）**：一个容器区域（原 `.scene`/`.screen` 所在区域），承载若干 piece。
- **piece**：台面上一段胶片对象。一个 piece 可以是「一整卷长条」或「剪出的单张」。
  - 每个 piece 有自己的位置 `{x, y}`（台面坐标）、所属卷 `rollId`、包含的帧范围、以及一个**自带的 canvas**（DOM 元素 `.piece` 内含 `<canvas>`）。
  - piece 渲染复用现有逐帧逻辑（片基 + 齿孔 + cover 裁帧 + 透光 glow）。长条 piece 内部仍可左右平移看更多帧（保留 `panX` 概念，但作用域收敛到单个 piece）。
- **导出**：遍历台面上所有 piece，按各自位置把它们的 canvas 合成到一张大画布 → **整个台面所见即所得**（含发光台面背景 `bg`）。

## 交互

- 从右侧 tray 把**一整卷**拖上台面 → 生成**一个可整体自由拖动的长条 piece**。
- 拖动任意 piece：一直跟随光标、自由 2D 摆放；被拖的 piece 抬到最上层（z-index）。用 pointer 事件实现（已有模式），不要用 HTML5 原生 drag（跟随/自由定位都差）。
- 长条 piece 超出台面：长条本身可整体拖走；在长条内部横向拖（或滚轮）可平移查看更多帧。需要区分「拖动整片移动」与「片内平移」两种手势（例如：拖 piece 边缘/外框 = 移动整片；在帧区横向拖 = 片内平移；或加一个移动把手）。实现时选一个清晰、不打架的方案并在 spec 留痕。
- tray、新建卷、卷内导入、缩略图、删除等现有交互保持可用。

## 阶段 1 — scene/piece 重构（需求 1）

把渲染从「单张满屏 canvas」改成「台面容器 + 每个胶片一个可自由拖动的 piece（DOM 元素含 canvas）」。

- 新状态：`pieces[]`，每项 `{ id, rollId, x, y, z, panX, ... }`。台面同时渲染所有 piece。
- 从 tray 拖一卷上台 = push 一个长条 piece（落点在光标处）。
- piece 自由拖动（pointer 事件，跟随光标，抬升 z-index）。
- 长条内部平移保留（作用域收敛到单 piece）。
- 导出改为「合成整个台面所有 piece + 发光背景」。
- 保留原有 tray/新建卷/导入/缩略图/删除/观片台预设轮播/底部规格 chip。
- 验收：能把同一卷或多卷拖上台面成多片、各自自由摆放、互不影响；导出图片 = 台面所见即所得。

## 阶段 2 — 按帧剪出单张（需求 4）

在长条 piece 上「按帧剪出整张」：

- 在某个长条 piece 上选中某一帧（点选/框选其中一帧），提供「剪下」操作 → 生成一个**独立单张 piece** 摆到台面，可自由拖动。
- 单张 piece 复用同一渲染（就是 N=1 的 piece），同样能导出、删除。
- 原长条是否保留该帧：默认保留（非破坏）；可留一个开关/交互让用户决定。实现时选简单方案并留痕。
- 验收：从一卷里剪出任意一帧成为独立单张，单独摆放并出现在导出里。

## 阶段 3 — 风格 + 设置拆分（需求 2、3）

需求 2（风格）：

- 去扁平化、去多余圆角，往真实灯箱/底片台质感走（更克制的圆角、更真实的材质/光影/边缘；台框、片基、齿孔、透光更接近实物）。
- 不要 neon 蓝紫描边那种「网页感」；保持现有暗房氛围但更"物理"。

需求 3（设置拆分）：

- 抽屉里把设置拆成两组，标题清晰分隔：
  - **观片台设置**：发光台面相关 —— 边缘透光感 glow、均匀度 unif、像素网格 pixel、台面色温 base、台框预设。
  - **胶片设置**：胶片本身相关 —— 底片圆角 radius、画幅规格（110/135/120/4×5）等。
- 现有滑块/色板/导出格式逻辑保留，仅重新归类与视觉分组。

## 通用验收

- 始终单文件、零依赖、可直接双击 `index.html` 打开运行。
- 已有功能不回退（新建卷、拖照片进卷、删除、轮播、规格切换、导出）。
- 移动端（`max-width:680px`）布局不破。
