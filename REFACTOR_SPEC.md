# 观片台重构 spec — 自由摆放的胶片台面

> 背景：当前 `index.html`（单文件，原生 HTML/CSS/Canvas）的模型是「一卷 = 铺满台面的一整张 `#film` canvas，只能左右 `panX` 平移，同一时刻只显示 activeRoll 一卷」。
> 目标：升级成「胶卷是桌上的实物，整屏自由摆放、像拼图一样拿在手上拖动」，观片台是一块固定的发光窗（= 导出区域），并借这次重构落地 4 个需求。
> 约束：**保持纯原生、零依赖、单文件 `index.html`**。不引入框架/构建链/canvas 库。复用现有 `films[]` 规格、`stripLayout()` 几何、`renderBg()`、逐帧 cover 裁切、`pointerdown/move/up + setPointerCapture` 拖动模式。

## 心智模型（真实灯箱）

观片台是**固定在中央的发光矩形窗（lightbox）**；胶卷是**桌上（整个屏幕）能滑动的实物**。你从右侧抽屉拿起一卷胶卷，它就跟在手上，放到桌上任意位置；要看长卷后面的帧，就把**整条胶卷滑过去**，让那几帧落到发光窗上。导出 = 给发光窗那块**拍一张**。

## 目标模型（术语）

- **桌面（desk）**：**整个屏幕/视口**，piece 的活动范围（不止观片台框内）。
- **观片台（lightbox）**：固定在中央的**发光矩形窗**，位置/大小固定，**就是导出区域**。不是 piece 的边界。
- **piece**：桌上一段胶片实物。一个 piece 可以是「一整卷长条」或「剪出的单张」。
  - 每个 piece 有自己的**视口坐标 `{x, y}`**、所属卷 `rollId`、包含的帧范围、一个**自带 canvas**（DOM 元素 `.piece` 内含 `<canvas>`）。
  - piece 渲染复用现有逐帧逻辑（片基 + 齿孔 + cover 裁帧 + 透光 glow）。**长条 piece 没有内部平移/滚动**——它就是一条完整实物，比屏幕宽就伸到屏幕外，靠**整条拖动**来移动。
- **导出**：以**观片台矩形**为画框，把背光底 + 所有 piece 按各自视口坐标相对观片台的偏移合成进去（露在框外的部分自然被裁掉）→ **导出 = 观片台那块区域的快照**。

## 交互（一体操作，无把手、无片内滚动）

- **拖出即跟随**：在右侧 tray 对某卷 `pointerdown`，**立刻**生成一个胶卷 piece 跟在光标上（不是松手后才出现），像把胶卷拿在手里。tray 里的卷是「库」，拿出来 = 在桌上放一个该卷的实例（库里的卷保留，可多次放）。
- **一体拖动**：直接抓胶卷**本体**就能拖，跟随光标、可摆到**整屏任意位置**（含观片台框外）；被拖的 piece 抬到最上层（z-index）。**不要**左上角把手、**不要**片内单独滚动——抓住就是移动整条，这是唯一动作。
- **拖出 tray 与桌上移动用同一套 pointer 机制**（统一手感）；全程用 pointer 事件，**不要** HTML5 原生 drag。
- **移除**：把 piece **拖回右侧 tray 区域**松手 = 从桌上拿走（回到库里），不再出现在导出。
- **长卷浏览** = 整条滑过去（见上，无滚动）。
- tray、新建卷、卷内导入、缩略图等现有交互保持可用。

## 阶段 1 — 桌面/piece 重构（需求 1）

> 注：当前分支已有一版 piece 重构，但交互方向偏了（活动范围被限在观片台内、用左上角把手移动、长条还有内部平移）。本阶段=按上面纠正后的「整屏自由摆 + 一体拖动 + 无把手 + 无片内滚动 + 拖回 tray 移除 + 导出只截观片台」改正。pieces[]/DOM+canvas/逐帧渲染这套底子保留复用。

- **piece 活动范围 = 整个视口**：`.pieces` 层铺满屏幕（不是只在观片台框内），piece 用视口坐标定位，可摆到观片台框外、可伸到屏幕外。
- **观片台是固定发光窗**：中央一块固定的发光矩形，仅作视觉与**导出画框**，不再是 piece 的容器边界。
- **拖出即跟随**：tray 里对某卷 `pointerdown` 立刻创建 piece 跟随光标；与桌上 piece 的移动**共用同一套 pointer 拖动逻辑**。tray 的卷是库，拿出 = 放实例（库保留）。
- **一体拖动，去掉把手与片内平移**：抓 piece 本体即移动整条，跟随光标、抬 z-index；删除原 `.piece-grip` 把手与单 piece 的 `panX` 内部平移逻辑。
- **拖回 tray 移除**：piece 拖到右侧 tray 区域松手 → 从 `pieces[]` 移除、DOM 移除。
- **导出改为「只截观片台矩形」**：以观片台在屏幕上的矩形为画框，离屏 canvas 画背光底 + 各 piece（按 `piece视口坐标 − 观片台视口坐标` 偏移），框外自然裁掉。
- 保留原有 tray/新建卷/导入/缩略图/观片台预设轮播/底部规格 chip。
- 验收：① 从 tray 拿卷时立刻有胶卷跟手；② 胶卷能摆到整屏任意处（含台框外）；③ 长卷靠整条滑动把不同帧滑到发光窗上；④ 没有把手、抓本体即拖；⑤ 拖回 tray 能移除；⑥ 导出图 = 观片台那块区域的快照（露在框外的胶片被裁掉）。

## 阶段 2 — 按帧剪出单张（需求 4）

在长条 piece 上「按帧剪出整张」：

- 在某个长条 piece 上选中某一帧（点选/框选其中一帧），提供「剪下」操作 → 生成一个**独立单张 piece** 摆到台面，可自由拖动。
- 单张 piece 复用同一渲染（就是 N=1 的 piece），同样能导出、删除。
- 原长条是否保留该帧：默认保留（非破坏）；可留一个开关/交互让用户决定。实现时选简单方案并留痕。
- 验收：从一卷里剪出任意一帧成为独立单张，单独摆放并出现在导出里。

## 阶段 3 — 风格 + 设置拆分（需求 2、3）

需求 2（风格）：

- 去扁平化、去多余圆角，往真实灯箱/底片台质感走（更克制的圆角、更真实的材质/光影/边缘；台框、片基、齿孔、透光更接近实物）。
- 不要 neon 蓝紫描边那种「网页感」；保持现有暗房氛围但更"物理"。

需求 3（设置拆分）：

- 抽屉里把设置拆成两组，标题清晰分隔：
  - **观片台设置**：发光台面相关 —— 边缘透光感 glow、均匀度 unif、像素网格 pixel、台面色温 base、台框预设。
  - **胶片设置**：胶片本身相关 —— 底片圆角 radius、画幅规格（110/135/120/4×5）等。
- 现有滑块/色板/导出格式逻辑保留，仅重新归类与视觉分组。
